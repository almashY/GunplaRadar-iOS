# GunplaRadar iOS - Claude 自動実行ルール

## ドキュメント自動更新

実装を変更・追加したとき、以下を自動で確認して必要であれば更新する。

- `docs/requirements/` — 機能仕様に変更があれば該当の F0X ファイルを更新
- `docs/detailed_design/` — クラス構成・シーケンス・画面設計に変更があれば更新
- `docs/basic_design/` — 対応方針・クラス図・シーケンス図に変更があれば更新
- `docs/tests/` — テストケースを追加・変更したとき、必ず該当の F0X ファイルを更新

## ユニットテストの自動判断と作成手順

新規クラス・メソッドを作成したとき、以下の基準でユニットテストが必要か判断する。

**テスト不要**
- SwiftUI の View（画面表示のみで完結するもの）
- ViewModel の単純なデータバインディング（表示更新のみ）

**テスト必要**
- ビジネスロジック（計算・変換・バリデーション・データ加工）を持つクラス・メソッド
- Repository の fetch/save/delete など副作用を持つ処理
- Service クラスのロジック

**作成手順（必ずこの順番で行う）**

1. テストが必要と判断したら、**先にテストケース一覧をユーザーに提示する**
2. 「このテストケースで作成しますか？（Yes / No）」と確認する
3. **Yes の場合のみ**テストコードと `docs/tests/` の更新を行う
4. **No の場合は**テストを作成しない（ユーザーの指示に従う）

## その他のルール

- 実装完了後、変更内容のサマリーを簡潔に出力する（変更ファイル一覧と要点）
- ドキュメントの更新は実装ファイルの変更と同じタイミングで行う（後回しにしない）
