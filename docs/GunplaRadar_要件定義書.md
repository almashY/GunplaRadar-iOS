# 要件定義書

## アプリ名
**Gunpla Radar**

---

## 1. 概要

### 1.1 目的
ガンプラ再販品の品出し日は店舗によって差異があり、再販予測日と実際の品出し日のずれを蓄積・分析することで、より効率的にガンプラを入手できる環境を提供する。また、再販日の記録や模型屋の管理を地図・カレンダーアプリとは独立して行えるツールとして機能させる。


### 1.2 対象ユーザー
ガンプラの再販情報を管理・追跡したいコレクター・購入者

### 1.3 開発環境
- プラットフォーム：iOS
- フレームワーク：SwiftUI

---

## 2. 機能一覧

| No | 機能名 | 概要 |
|----|--------|------|
| F01 | ほしい物リスト | ガンプラ情報の登録・管理・ソート・検索 |
| F02 | 再販カレンダー | 再販日・巡回予定のカレンダー表示と管理 |
| F03 | 店舗分析 | 店舗登録・お気に入り管理・品出しずれ分析 |
| F04 | ガンプラ巡回 | 巡回予定の作成・管理・通知 |
| F05 | アプリ設定 | 通知設定・蓄積データ管理 |

## 3. 機能詳細

### F01 ほしい物リスト

#### 3.1.1 新規登録
登録ボタン押下により、以下の情報を入力可能とする。

| 項目 | 必須 | 備考 |
|------|------|------|
| 画像 | - | カメラ撮影またはライブラリから選択 |
| URL | - | Amazon・プレバン等 |
| 商品名 | ○ | |
| グレード | ○ | RE / PG / MG / RG / HG / EG / その他 の固定リストからドラムロールで選択。「その他」選択時はテキスト入力 |
| 値段 | - | |
| 次回再販日 | - | 再販カレンダーと連携。入力欄タップでカレンダーUIを表示して選択 |
| 発売日 | - | 入力欄タップでカレンダーUIを表示して選択 |
| 購入日 | - | |
| 購入場所 | - | 店舗分析機能と連携 |
| 優先度 | ○ | 最高・高・中・低の4段階 |
| タグの色 | - | |

日付入力欄（発売日・再販日）はテキスト直接入力ではなく、タップ操作でDatePickerダイアログ（カレンダーUI）を表示し、日付を選択して入力する。

#### 3.1.2 過去情報からの作成
「過去の情報から作成」ボタン押下で登録済み情報を検索・選択可能。
選択すると以下の基本情報が自動入力される。

- 画像
- URL
- 商品名
- グレード
- 値段
- 発売日
- 購入日

#### 3.1.3 ソート機能
複数ソートに対応し、左から順に優先度が高い。

| ソートキー | 備考 |
|-----------|------|
| 優先度順 | |
| 再販日順 | 現在日時に最も近い順。再販日が過去のものは非表示 |
| 発売日順 | |
| グレード順 | |
| 値段順 | |

#### 3.1.4 検索機能
商品名・グレード・タグ等をキーワードで絞り込み検索できる。

#### 3.1.5 アイテム詳細表示
リストのアイテムをタップすると詳細画面（S02a）へ遷移し、以下の情報を確認できる。

| 項目 | 備考 |
|------|------|
| 商品名・グレード | タグカラーとともに表示 |
| 価格・優先度 | |
| 発売日・再販日・購入日 | |
| URL | タップでブラウザ遷移 |

詳細画面右上の「編集」ボタンで編集画面へ遷移し、編集画面右上の「保存」ボタンで保存する。

### F02 再販カレンダー

#### 3.2.1 カレンダー表示
- ほしい物リストに登録した再販日をカレンダー上に表示する。
- 巡回予定日は赤色で表示する。

#### 3.2.2 日付タップ操作
カレンダーの任意の日付をタップすると、ボトムシートが表示される。
ボトムシート内には以下を表示する。

- その日に再販日が登録されているアイテム一覧
- 「新しい再販予定日を作成」ボタン

#### 3.2.3 再販予定日の新規作成
「新しい再販予定日を作成」ボタン押下で以下が可能。

- 基本情報の新規入力
- ほしい物リストからの選択

作成完了後、再販カレンダー画面に遷移して登録内容を反映する。

#### 3.2.4 品出し差分登録
「再販予定日との差分登録」ボタン押下で日にち選択モードに移行。

1. 再販予定日が登録されている日付を選択
2. 対象タグをタップして日にち登録画面に遷移
3. 品出し日時・品出し店舗を入力して登録
4. 再販予定日と品出し日時の差分を算出し、店舗情報に紐付けて保存

**差分算出ロジック例**
- 再販予定日：6日
- 品出し日時：8日 午前11時
- 差分：2日11時間

この差分データを店舗ごとに蓄積し、平均ずれ時間として保持する。

### F03 店舗分析

#### 3.3.1 画面構成
- 上部：検索バー（場所・住所・店名で検索）
- 中部：マップ表示（ガンプラ取扱店舗をピン表示）
- 右下：店舗一覧ボタン

#### 3.3.2 検索機能
`MKLocalSearchCompleter` によるリアルタイム入力補完に対応する。

- 文字入力中に候補リストを表示（地名・住所・店舗名・POI名すべて対応）
- 候補をタップするとその地点にカメラが移動し、場所詳細シートを表示する

#### 3.3.3 店舗登録
検索または地図上のPOIタップから店舗を選択し、ガンプラ取扱店舗として登録する。
手動での緯度・経度入力による登録は廃止し、以下の方法のみ対応する。

| 登録方法 | 操作 |
|----------|------|
| 検索から登録 | 候補タップ後に表示される詳細シートの「ガンプラ巡回店舗に追加」ボタン |
| POIタップから登録 | マップ上のPOI（コンビニ・ショッピングモール等）をタップ後に表示される詳細シートの「ガンプラ巡回店舗に追加」ボタン |

すでに登録済みの店舗の場合は「登録済み」と表示し、重複登録を防ぐ。

#### 3.3.4 POIタップ機能
iOSマップと同様に、マップ上の既存POIアイコンをタップすると詳細シートが表示される。

表示内容：
- 店舗名・住所
- 電話番号（タップで発信）
- Webサイト（タップでブラウザ遷移）
- 「ガンプラ巡回店舗に追加」ボタン

#### 3.3.5 登録済み店舗詳細表示
登録済み店舗のピンをタップするとボトムシートが表示され、以下を確認できる。

- 店舗基本情報（名前・座標）
- 再販予定日からの平均ずれ時間
- お気に入り登録・解除ボタン
- 削除ボタン

#### 3.3.6 お気に入り機能
- お気に入り登録するとマップ上にマークが表示される。
- ほしい物リストの購入場所でお気に入り店舗から自動検索できる。

#### 3.3.7 店舗一覧
「店舗一覧」ボタン押下でお気に入り店舗のリストを表示。
店舗を選択すると以下が可能。

- お気に入り登録・解除
- 再販予定日からの平均ずれ確認

### F04 ガンプラ巡回

#### 3.4.1 巡回予定の作成
「巡回予定を作成」ボタン押下で以下の情報を入力する。

| 項目 | 備考 |
|------|------|
| 日にち | 入力欄タップでカレンダーUIを表示して選択 |
| 時刻 | テキスト入力（HH:mm形式） |
| 店舗名 | 店舗分析の登録店舗から選択可 |
| 欲しいガンプラ | ほしい物リストから選択可 |

保存後、再販カレンダーに巡回予定として登録される。

#### 3.4.2 巡回予定一覧
- デフォルトは作成順に表示。
- リストをタップして詳細を閲覧可能。

#### 3.4.3 巡回詳細表示
再販カレンダーの巡回予定日タップまたはリストタップで詳細を表示。

**画面構成**
- 上部：日にち
- 中部：欲しいガンプラ一覧
- 下部：時間 / 店舗名 / 予定差分（店舗分析の平均ずれ）を時間順に表示

#### 3.4.4 通知
巡回予定日当日に通知を受け取ることができる（設定でON/OFF可能）。

### F05 アプリ設定

| 設定項目 | 内容 |
|----------|------|
| 通知設定 | 巡回予定通知のON/OFF切り替え |
| 蓄積データ管理 | 店舗分析の品出し差分データを削除可能。一括削除または編集モードで複数選択削除に対応。 |
