# F04 ガンプラ巡回 - 対応方針

## ドメイン概要

巡回予定の作成・管理と当日通知を担うドメイン。
`PatrolPlan` が中心エンティティであり、`GunplaItem`・`Store` を参照して巡回に必要な情報を集約する。
通知スケジューリングはインフラ層（UNUserNotificationCenter）の責務とし、ドメインは通知の有効・無効フラグのみを保持する。

---

## ドメインの責務

| 責務 | 担当クラス |
|------|-----------|
| 巡回予定の保持・管理 | `PatrolPlan`（エンティティ） |
| 巡回詳細情報の組み立て | `PatrolPlanService`（ドメインサービス） |
| 巡回予定の保存・取得 | `PatrolPlanRepository`（リポジトリ） |

---

## 方針

### 巡回予定の作成
- `PatrolPlan` は日付・時刻・店舗ID・対象アイテムIDリスト・通知フラグを持つ。
- 保存完了後、`PatrolPlan.date` を再販カレンダーに反映させる（F02連携）。

### 巡回詳細表示
- 詳細画面では `PatrolPlan` に紐付いた `GunplaItem` と `Store` を取得して表示する。
- 予定差分（店舗の平均ずれ時間）は `Store.averageDelayHours` をそのまま表示する。
- `PatrolPlanService` が `PatrolPlan` + `GunplaItem` + `Store` を組み合わせた詳細情報を返す。

### 通知
- `PatrolPlan.notifyEnabled` が true の場合のみ通知をスケジュールする。
- 通知スケジュールの登録・解除はインフラ層（`UNUserNotificationCenter` ラッパー）に委譲し、ドメインはフラグ管理のみを行う。
- F05（アプリ設定）でグローバルに通知をOFFにした場合は全 `PatrolPlan` の通知を無効化する。

---

## 他機能との連携

| 連携先 | 内容 |
|--------|------|
| F01 ほしい物リスト | 巡回対象アイテムとして `GunplaItem` を参照 |
| F02 再販カレンダー | `PatrolPlan.date` をカレンダーに赤色表示 |
| F03 店舗分析 | 巡回先の `Store` を参照し平均ずれ時間を表示 |
| F05 アプリ設定 | グローバル通知設定が変更された際に通知スケジュールを更新 |
