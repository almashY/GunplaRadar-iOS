# F02 再販カレンダー - 対応方針

## ドメイン概要

再販予定日の可視化と、実際の品出し日との差分を記録・蓄積するドメイン。
カレンダー表示は `GunplaItem.restockDate` を集約して行い、差分記録は `StockDelayRecord` として独立したエンティティで管理する。

---

## ドメインの責務

| 責務 | 担当クラス |
|------|-----------|
| 再販日のカレンダー集約 | `RestockCalendarService`（ドメインサービス） |
| 品出し差分の記録 | `StockDelayRecord`（エンティティ） |
| 差分時間の算出 | `StockDiffCalculator`（ドメインサービス） |
| 差分記録の保存・取得 | `StockDelayRecordRepository`（リポジトリ） |

---

## 方針

### カレンダー表示
- カレンダー表示用のデータは `GunplaItemRepository` から取得した `restockDate` を日付キーで集約する。
- `PatrolPlan` の日付も同時に集約し、巡回予定日を赤色表示するためのフラグを付与する。
- 表示ロジックはドメインサービスが日付マップとして返すことでViewの責務を減らす。

### 品出し差分登録
- 差分算出は `再販予定日` と `実際の品出し日時` の差を時間単位で算出し `StockDelayRecord` に保持する。
- 算出ロジックをドメインサービス `StockDiffCalculator` に集約し、算出ルールの変更を局所化する。
- 登録完了後、該当店舗の `averageDelayHours` を再計算して更新する。

### 再販予定の新規作成
- カレンダーから新規作成する場合は、タップした日付を `restockDate` の初期値として `GunplaItem` の登録フローへ引き渡す。

---

## 他機能との連携

| 連携先 | 内容 |
|--------|------|
| F01 ほしい物リスト | `GunplaItem.restockDate` を参照してカレンダーに表示 |
| F03 店舗分析 | 差分登録時に `Store` を指定し、`StockDelayRecord` を紐付け |
| F04 ガンプラ巡回 | `PatrolPlan` の日付を赤色表示するために参照 |
