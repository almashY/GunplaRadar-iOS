# エラーハンドリング設計書

---

## 基本方針

- エラーは `RepositoryError` / `ValidationError` に分類して `throws` で伝播させる
- ViewModel がエラーを catch し、`errorMessage: String?` プロパティに格納する
- View は `errorMessage` を監視し、非 nil になったらアラートを表示する
- ユーザーに見せるメッセージは日本語で統一し、技術的な詳細は含めない

---

## エラー型定義

```swift
enum RepositoryError: Error {
    case saveFailed
    case deleteFailed
    case fetchFailed
    case notFound
}

enum ValidationError: Error {
    case requiredFieldMissing(field: String)
    case invalidFormat(field: String)
}
```

---

## エラー分類と対応方針

| エラー種別 | 発生箇所 | ユーザーへの表示 | リトライ |
|-----------|---------|----------------|---------|
| `RepositoryError.saveFailed` | 全リポジトリの save | 「保存に失敗しました。もう一度お試しください。」| アラートに「再試行」ボタンを表示 |
| `RepositoryError.deleteFailed` | 全リポジトリの delete | 「削除に失敗しました。もう一度お試しください。」| アラートに「再試行」ボタンを表示 |
| `RepositoryError.fetchFailed` | 全リポジトリの fetch | 「データの取得に失敗しました。」| アラートに「再読み込み」ボタンを表示 |
| `RepositoryError.notFound` | fetch(by id:) で対象なし | 「対象のデータが見つかりませんでした。」| なし（OKボタンのみ） |
| `ValidationError.requiredFieldMissing` | 登録・編集フォーム保存時 | 「{field}は必須項目です。」| なし（フォームに戻る） |
| `ValidationError.invalidFormat` | 時刻入力（HH:mm） | 「{field}の形式が正しくありません。」| なし（フォームに戻る） |
| 通知スケジュール失敗 | `NotificationScheduler.schedule` | 「通知の設定に失敗しました。設定アプリで通知の許可を確認してください。」| なし |
| MapKit 検索失敗 | `MKLocalSearch.start` | 「店舗の検索に失敗しました。通信環境を確認してください。」| アラートに「再試行」ボタンを表示 |

## ViewModel での共通パターン

```swift
func save(item: GunplaItem) {
    Task {
        do {
            try repository.save(item)
        } catch ValidationError.requiredFieldMissing(let field) {
            errorMessage = "\(field)は必須項目です。"
        } catch {
            errorMessage = "保存に失敗しました。もう一度お試しください。"
        }
    }
}
```

---

## View でのアラート表示パターン

```swift
.alert("エラー", isPresented: Binding(
    get: { viewModel.errorMessage != nil },
    set: { if !$0 { viewModel.errorMessage = nil } }
)) {
    Button("OK") { viewModel.errorMessage = nil }
} message: {
    Text(viewModel.errorMessage ?? "")
}
```

## 通知権限未許可の扱い

巡回予定登録時に通知権限が未許可の場合、`notifyEnabled` は `false` のまま保存し、ユーザーに「通知が許可されていません。設定アプリから許可してください。」と案内する。アプリ側から強制的に設定画面へ遷移はしない。
