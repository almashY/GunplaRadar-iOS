# 通知設計書

使用フレームワーク：**UserNotifications**（ローカル通知）

---

## 通知の種類

| 種別 | トリガー | 対象機能 |
|------|---------|---------|
| 巡回予定当日通知 | 巡回予定の当日 `time` に発火 | F04 ガンプラ巡回 |

---

## 通知内容

| 項目 | 内容 |
|------|------|
| タイトル | 「本日の巡回予定」 |
| 本文 | 「{店舗名} への巡回予定があります。」 |
| 識別子 | `patrol-{PatrolPlan.id.uuidString}` |

---

## 通知スケジュールのタイミング

| 操作 | 処理 |
|------|------|
| 巡回予定を作成（notifyEnabled = true） | `schedule(for:)` を呼び出し、当日 `time` に発火する通知を登録 |
| 巡回予定を削除 | `cancel(planId:)` で通知を解除 |
| アプリ設定の通知を OFF | `cancelAll()` で全通知を解除 |
| アプリ設定の通知を ON | `notifyEnabled = true` の巡回予定を全件 `schedule(for:)` で再登録 |

---

## NotificationScheduler 実装仕様

```swift
final class NotificationScheduler {

    func schedule(for plan: PatrolPlan) throws {
        // 1. 通知許可チェック
        // 2. DateComponents を plan.date + plan.time から構築
        // 3. UNCalendarNotificationTrigger を作成
        // 4. UNNotificationRequest を identifier: "patrol-{plan.id}" で登録
    }

    func cancel(planId: UUID) {
        UNUserNotificationCenter.current()
            .removePendingNotificationRequests(
                withIdentifiers: ["patrol-\(planId.uuidString)"]
            )
    }

    func cancelAll() {
        UNUserNotificationCenter.current().removeAllPendingNotificationRequests()
    }
}
```

---

## 通知権限リクエスト

| タイミング | 処理 |
|-----------|------|
| 初回起動時 | `UNUserNotificationCenter.requestAuthorization(options: [.alert, .sound, .badge])` を呼び出す |
| 拒否された場合 | 通知機能を使わずアプリは起動可能。巡回予定登録時に権限なしを検知したら「設定アプリから許可してください」と案内 |

---

## グローバル通知設定と個別通知設定の関係

```
実際に通知が届く条件:
  AppSettings.notifyEnabled = true
  AND
  PatrolPlan.notifyEnabled  = true
```

- `AppSettings.notifyEnabled` を OFF にすると全スケジュールを解除する
- ON に戻した場合、`PatrolPlan.notifyEnabled = true` のものだけ再登録する
- `PatrolPlan.notifyEnabled` は `AppSettings` の変更では変更しない（個別の状態を保持）
